## Overview The `Icon` component provides a consistent way to display Font Awesome icons throughout your application. It automatically converts PascalCase icon names to Font Awesome classes and supports various sizes, colors, and styling options. ## Usage ```html ``` ## Live Examples ### Basic Icons ### Size Variations **Small Icons:** **Medium Icons:** **Large Icons:** ### Color Variations **Theme Colors:** **Custom Colors:** **Status Colors:** ### Action Icons ## Attributes | Attribute | Type | Required | Default | Description | |-----------|------|----------|---------|-------------| | `icon` | string | Yes | - | Icon name in PascalCase (e.g., "Home", "Search", "User") | | `color` | string | No | - | Icon color (CSS color value) | | `size` | string | No | `medium` | Icon size: `small`, `medium`, or `large` | ## How It Works ### Icon Name Conversion 1. **PascalCase Input**: Takes icon names like "Home", "Search", "User" 2. **Font Awesome Mapping**: Converts to Font Awesome classes like "fa-home", "fa-search", "fa-user" 3. **Special Cases**: Handles Material Design icon names with custom mappings 4. **Fallback**: Uses the original name if no mapping is found ### Size Handling 1. **Size Attribute**: Processes `size` attribute (small, medium, large) 2. **CSS Classes**: Applies appropriate CSS classes for consistent sizing 3. **Button Context**: Adjusts sizing when used inside buttons 4. **Custom Sizing**: Supports custom sizing via sx: attributes ### Color Processing 1. **Color Attribute**: Processes `color` attribute for direct color values 2. **SX Color**: Supports `sx:color` for CSS variable references 3. **Theme Integration**: Works with theme color variables 4. **Fallback**: Uses default text color if no color is specified
