## Overview The `Navbar` component provides a dynamic navigation header that automatically updates based on the current application path. It displays a formatted title derived from the active path and includes a right-side area for additional actions or content. The component subscribes to path changes and updates its display accordingly, making it ideal for application-wide navigation headers. ## Features - **Dynamic Title**: Automatically updates based on current path - **Path Formatting**: Converts technical paths to user-friendly titles - **Component Name Cleanup**: Removes "x-" prefix from component names - **State Subscription**: Automatically updates when path changes - **Flexible Layout**: Supports additional content in the right area - **Responsive Design**: Adapts to different screen sizes - **Theme Integration**: Consistent styling with framework theme ## How It Works ### Path Processing 1. **State Subscription**: Subscribes to `activePath` state changes 2. **Path Parsing**: Splits the path into segments 3. **Title Generation**: Converts path segments to readable titles 4. **Component Cleanup**: Removes "x-" prefixes from component names 5. **Display Update**: Updates the navbar title with formatted text ### Title Formatting 1. **Segment Processing**: Converts each path segment to title case 2. **Separator Addition**: Adds " / " between segments 3. **Special Handling**: Handles root path ("/") as "Home" 4. **Component Names**: Removes "x-" prefix from component names 5. **Final Display**: Shows formatted title in the navbar ### State Management 1. **Automatic Updates**: Updates when `activePath` changes 2. **Initial Load**: Displays current path on component creation 3. **Real-time Sync**: Stays in sync with navigation changes 4. **Error Handling**: Gracefully handles invalid or missing paths ## Best Practices ### Content Design - Keep navbar content concise and focused - Use consistent styling across all navbar instances - Test navbar with different path lengths and formats - Consider mobile responsiveness for navbar content ### Performance - Avoid complex content that could slow down rendering - Use efficient state subscriptions - Test navbar performance with frequent path changes - Consider caching for static navbar content ### Accessibility - Ensure sufficient color contrast for navbar text - Provide meaningful labels for interactive elements - Test with screen readers and keyboard navigation - Use semantic HTML structure when possible ### Responsive Design - Test navbar on different screen sizes - Consider hiding or simplifying content on mobile - Use responsive styling for different breakpoints - Ensure touch-friendly button sizes on mobile ##
